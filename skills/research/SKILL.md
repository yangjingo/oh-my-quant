# research

编排完整量化研究流程，输出综合研究报告。

## 路由规则

- 单一任务 → 直接调用对应子 skill（data / factor / backtest / risk）
- 多步骤任务 → 使用本 skill 编排
- 用户说"帮我研究 X 策略" / "完整分析 Y" → 使用本 skill

## 标准流程

```
1. data    → 获取数据
2. factor  → 因子分析
3. backtest → 策略回测
4. risk    → 风险评估
5. 综合报告
```

## 第 0 步：对齐目标

从用户输入中提取：
- 研究对象（标的/市场/周期）
- 研究假设（要验证什么）
- 成功标准（IC 阈值、夏普下限等）
- 时间范围（训练期/样本外）

如果任何一项缺失，追问用户确认。

## 第 1-4 步：依次调用子 skill

按每个子 skill 的规范执行，结果保留在对应目录：
```
research/{YYYY-MM-DD}_{研究主题}/
├── data/         # 数据文件
├── factor/       # 因子分析输出
├── backtest/     # 回测结果
└── risk/         # 风险评估
```

## 第 5 步：综合报告

```markdown
# 综合研究报告：{主题}

## 摘要
- 目标 / 核心发现 / 决策（可行/需改进/不可行）

## 数据
- 范围、样本量、质量问题

## 因子
- IC Mean / ICIR / 分层单调性
- 与其他因子相关性

## 回测
| 指标 | 策略 | 基准 | 差值 |
|------|------|------|------|
| 年化 | ... | | |
| 夏普 | ... | | |
| 最大回撤 | ... | | |

## 风险
- 主要风险点 / 压力测试 / 尾部特征

## 样本外
- 衰减评估

## 改进方向
- 短期 (1-2d) / 中期 (1-2w) / 不建议
```
